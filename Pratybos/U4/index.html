    U4: Gilus Kopijavimas (Deep Copy)

    Savaitės: 7-8
    Svoris: 1 balas
    Terminas: Savaitės 8 pabaiga

    📚 Prieš pradedant

    🎯 Užduoties tikslas

    Išmokti teisingai kopijuoti objektus, kurie turi dinaminius resursus (rodykles). Suprasti shallow copy vs deep copy problemą. Įgyvendinti Rule of Three (destruktorius, copy constructor, copy assignment operator).

    Atlikę šią užduotį, mokėsite:

    • ✅ Suprasti shallow copy problemą (dangling pointer)
    • ✅ Rašyti copy constructor su deep copy
    • ✅ Rašyti copy assignment operator (operator=)

    • ✅ Testuoti kopijavimą su pass-by-value
    • ✅ Debug'inti memory problemas su logging

    💡 Kas yra Shallow vs Deep Copy?

    Shallow Copy (Seklus kopijavimas):

    Default behavior - C++ kompiliatorius automatiškai sukuria: 

    Kambarys k2 = k1;  // Default copy constructor

    Problema: Kopijuoja tik rodyklės reikšmę (memory address), ne objektą!

    k1:  durys → [Durys objektas @0x1234]
              ↑
k2:  durys ──┘  (ta pati rodyklė!)

    Rezultatas: - Abu objektai dalijasi tuo pačiu Durys objektu - Destruktorius bando delete du kartus → CRASH! 💥

    Deep Copy (Gilus kopijavimas):

    Mūsų sprendimas: Kopijuoti patį objektą, ne tik rodyklę!

    k1:  durys → [Durys objektas @0x1234]

k2:  durys → [Durys objektas @0x5678]  (NAUJA kopija!)

    Rezultatas: - Kiekvienas objektas turi savo Durys kopiją - Destruktorius delete skirtingus objektus → SAFE! ✅

    💡 Rule of Three

    Jei jūsų klasė turi bent vieną iš šių: 1. Destruktorius 2. Copy constructor 3. Copy assignment operator

    ...tai privalote turėti visus tris!

    Kodėl? - Destruktorius → klasė valdo resursus - Valdo resursus → reikia deep copy - Reikia deep copy → reikia copy constructor IR copy assignment

    Kambarys klasei: 1. ✅ ~Kambarys() - destruktorius (turėjome U3) 2. ✅ Kambarys(const Kambarys&) - copy constructor (pridėsime U4) 3. ✅ Kambarys& operator=(const Kambarys&) - copy assignment (pridėsime U4)

    1 žingsnis: Shallow copy problema (demo)

    Direktorija: U4/01/

    Reikalavimai:

    1. Kopijuokite U3/03/ kodą į U4/01/
    2. Visos klasės: Langas, Durys, Kambarys

    3. Visi failai: .h, .cpp, skaiciavimai.*, main.cpp, Makefile

    4. Sukurkite globalią funkciją skaiciavimai.h/.cpp: 

      void testuotiKopija(Kambarys k);  // Pass-by-VALUE (kopija!)

    Funkcijos turinys: 

    void testuotiKopija(Kambarys k) {
    cout << "Funkcijoje: sienu plotas = " 
         << apskaiciuotiSienuPlota(k) << endl;
}

    1. Main funkcijoje - demonstruoti problemą: 
      // Sukurti kambarį su langais ir durimis
Kambarys* k1 = new Kambarys(5.0, 4.0, 3.0);
// ... pridėti langus, duris ...

// TESTAS 1: Pirmas kvietimas (veiks)
cout << "Pirmas kvietimas:" << endl;
testuotiKopija(*k1);

// TESTAS 2: Antras kvietimas (CRASH arba blogas rezultatas!)
cout << "Antras kvietimas:" << endl;
testuotiKopija(*k1);  // ← PROBLEMA čia!

delete k1;

    Rezultatas: - 1-as kvietimas: Veikia OK - 2-as kvietimas: CRASH arba neteisingi rezultatai - Kodėl? Shallow copy → dangling pointer!

    Pavyzdys (galimas išvesties fragmentas): 

    Pirmas kvietimas:

[DEBUG] Kambarys naikinamas
[DEBUG] Kambarys naikina savo duris
[DEBUG] Durys sunaikintos. Liko duriu: 0  ← delete durys!

Antras kvietimas:

    Užduotis studentui: - ✅ Paleisti programą - ✅ Pamatyti CRASH arba blogus rezultatus - ✅ Suprasti problemą (shallow copy) - ✅ README.md - aprašyti, kas nutiko

    2 žingsnis: Copy constructor

    Direktorija: U4/02/

    Reikalavimai:

    1. Pridėti copy constructor į Kambarys.h: 

      // Copy constructor
Kambarys(const Kambarys& kitas);

    2. Implementuoti copy constructor Kambarys.cpp: 

      Kambarys::Kambarys(const Kambarys& kitas) 
    : ilgis(kitas.ilgis), 
      plotis(kitas.plotis), 
      aukstis(kitas.aukstis) {

    cout << "[DEBUG] Kambarys kopijuojamas (copy constructor)" << endl;

    // 1. Kopijuoti langus (vector daro deep copy automatiškai!)
    langai = kitas.langai;

    // 2. Kopijuoti duris (DEEP COPY - naujas objektas!)
    if (kitas.durys != nullptr) {
        durys = new Durys(*kitas.durys);  // Sukurti NAUJĄ Durys objektą
    } else {
        durys = nullptr;
    }
}

    3. Testuoti su ta pačia testuotiKopija() funkcija: 

      // Dabar turėtų veikti!
cout << "Pirmas kvietimas:" << endl;
testuotiKopija(*k1);

cout << "Antras kvietimas:" << endl;
testuotiKopija(*k1);  // ← Dabar OK!

    Rezultatas: - ✅ Abu kvietimai veikia! - ✅ Logging rodo copy constructor kvietimus - ✅ Kiekvienas Kambarys turi savo durų kopiją

    3 žingsnis: Copy assignment operator (Rule of Three)

    Direktorija: U4/03/

    Reikalavimai:

    1. Pridėti copy assignment operator į Kambarys.h: 

      // Copy assignment operator
Kambarys& operator=(const Kambarys& kitas);

    2. Implementuoti operator= Kambarys.cpp: 

      Kambarys& Kambarys::operator=(const Kambarys& kitas) {
    cout << "[DEBUG] Kambarys kopijuojamas (operator=)" << endl;

    // 1. Patikrinti self-assignment
    if (this == &kitas) {
        cout << "[DEBUG] Self-assignment aptiktas, praleidziama" << endl;
        return *this;
    }

    // 2. Atlaisvinti senus resursus
    if (durys != nullptr) {
        cout << "[DEBUG] Atlaisvinamos senos durys" << endl;
        delete durys;
        durys = nullptr;
    }
    langai.clear();  // Išvalyti langų vektorių

    // 3. Kopijuoti naujus duomenis
    ilgis = kitas.ilgis;
    plotis = kitas.plotis;
    aukstis = kitas.aukstis;

    langai = kitas.langai;  // Vector deep copy

    if (kitas.durys != nullptr) {
        durys = new Durys(*kitas.durys);  // Deep copy
    } else {
        durys = nullptr;
    }

    return *this;
}

    3. Testuoti su assignment: 

      Kambarys k1(5.0, 4.0, 3.0);
// ... pridėti langus, duris ...

Kambarys k2(3.0, 3.0, 2.5);

// Assignment
cout << "--- Priskyrimas: k2 = k1 ---" << endl;
k2 = k1;

// Testuoti k2
cout << "k2 sienu plotas: " << apskaiciuotiSienuPlota(k2) << endl;

// Self-assignment testas
cout << "--- Self-assignment: k2 = k2 ---" << endl;
k2 = k2;  // Turėtų aptikti ir praleisti

    4. Ekstremalus testas: 

      Kambarys k1(5.0, 4.0, 3.0);
// ... setup ...

Kambarys k2(3.0, 3.0, 2.5);
k2 = k1;

// Delete k1, bet k2 turėtų veikti!
delete &k1;  // (jei dynamic)

cout << "Po k1 delete, k2 sienu plotas: " 
     << apskaiciuotiSienuPlota(k2) << endl;  // ← Turėtų veikti!

    Rezultatas: - ✅ Copy assignment veikia - ✅ Self-assignment aptiktas - ✅ k2 veikia net po k1 sunaikinimo (deep copy!)

## 💭 Pagrindinės įžvalgos

1. Shallow copy → dangling pointer → crash
2. Deep copy → naujas objektas → safe
3. Rule of Three: destruktorius + copy constructor + operator=
4. Self-assignment check svarbus!

    ❓ Dažnai užduodami klausimai

    K: Kodėl reikia if (this == &kitas) operator= ?
    A: Self-assignment (k = k) be šios patikros gali sunaikinti objektą prieš kopijuojant!

    K: Kodėl return *this operator= ?
    A: Kad veiktų chaining: a = b = c (dešinė į kairę).

    K: Ar vector automatiškai daro deep copy?
    A: Taip! langai = kitas.langai - vector kopijuoja visus elementus.

    K: Kodėl new Durys(*kitas.durys)?
    A: *kitas.durys - dereferencing (gauti objektą), tada Durys copy constructor.

    K: Ar reikia langai.clear() operator= ?
    A: Ne (assignment perkraus), bet gera praktika (explicit cleanup).
